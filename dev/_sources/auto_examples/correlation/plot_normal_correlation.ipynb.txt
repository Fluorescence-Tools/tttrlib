        "\n# Normal multi-tau correlation\n\nAn alternative approach is to calculate the FCS curve by software\non the basis of the directly recorded asynchronous photon count data.\nThe straightforward approach would be to generate a continuous file of\nfluorescence intensity values in consecutive time bins of fixed bin\nwidth, and to use this file for a conventional autocorrelation calculation.\nBut then again, one encounters the same unreasonably large amount of data\nas when directly recording fluorescence intensity values in a synchronous\ndata acquisition mode.\n\n\nWhen applying the algorithm only at the increasingly spaced lag\ntimes as given by Eq. (2), it will completely miss, e.g., the strong\nautocorrelation of any periodic signal with a repetition time not\nincluded within the vector of used lag times. To avoid that, one usually\napplies an averaging procedure by coarsening the time resolution of the photon\ndetection times tj when coming to the calculation of the autocorrelation\nfunction at increasingly larger lag time. This is equivalent to the multiple-tau\nand multiple-sampling time correlation method employed in hardware\ncorrelators [12].\n\n:cite:`Wahl2003`.\n"
