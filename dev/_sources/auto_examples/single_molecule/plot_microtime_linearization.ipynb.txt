        "def create_example_linearization_table(n_channels=256):\n    \"\"\"\n    Create an example linearization table.\n\n    In real use, these tables would come from calibration measurements.\n    This example creates a slightly non-linear response to demonstrate the effect.\n    \"\"\"\n    # Create a slightly non-linear response (quadratic distortion)\n    channels = np.arange(n_channels, dtype=np.float32)\n\n    # Non-linear response: quadratic distortion\n    a = 1.0\n    b = 0.0001  # Small quadratic term\n    linearized = a * channels + b * channels**2\n\n    return linearized.astype(np.float32)\n\n# Create linearization tables for different channels\nlt_ch0 = create_example_linearization_table(256)  # Channel 0: 256 channels\nlt_ch8 = create_example_linearization_table(256)  # Channel 8: 256 channels\n\nprint(f\"Linearization table for channel 0 shape: {lt_ch0.shape}\")\nprint(f\"Linearization table for channel 8 shape: {lt_ch8.shape}\")"

        "# Example 1: Post-Read Linearization\n\nApply linearization to already-loaded TTTR data.\n\n"

        "# Create synthetic TTTR data for demonstration\nn_photons = 10000\nmacro_times = np.arange(n_photons, dtype=np.uint64) * 100  # 100 time units apart\nmicro_times = np.random.randint(0, 256, n_photons, dtype=np.uint16)\nrouting_channels = np.random.randint(0, 16, n_photons, dtype=np.int8)\nevent_types = np.zeros(n_photons, dtype=np.int8)\n\nprint(f\"\\nCreated synthetic TTTR data:\")\nprint(f\"  Photons: {n_photons}\")\nprint(f\"  Macro time range: {macro_times[0]} - {macro_times[-1]}\")\nprint(f\"  Micro time range: {micro_times.min()} - {micro_times.max()}\")\nprint(f\"  Routing channels: {np.unique(routing_channels)}\")\n\n# Create TTTR object from the synthetic data\ntttr = tttrlib.TTTR(\n    macro_times,\n    micro_times,\n    routing_channels,\n    event_types\n)\n\nprint(f\"\\nTTTR object created with {tttr.size()} events\")\n\n# Get microtime histogram before linearization\nhist_before, bins_before = tttr.get_microtime_histogram()\nprint(f\"Microtime histogram before linearization: {len(hist_before)} bins\")\n\n# Configure linearization for specific channels\nprint(\"\\nConfiguring linearization...\")\n\n# Create linearizer\nlinearizer = tttrlib.MicrotimeLinearization()\n\n# Set LUTs for specific channels\nlut_ch0 = tttrlib.VectorFloat()\nfor val in lt_ch0:\n    lut_ch0.append(val)\nlinearizer.set_channel_lut(0, lut_ch0)  # Channel 0\n\nlut_ch8 = tttrlib.VectorFloat()\nfor val in lt_ch8:\n    lut_ch8.append(val)\nlinearizer.set_channel_lut(8, lut_ch8)  # Channel 8\n\n# Set shifts for channels (optional)\nlinearizer.set_channel_shift(0, 0)   # No shift for channel 0\nlinearizer.set_channel_shift(8, 5)   # Shift channel 8 by 5\n\n# Configure the TTTR object with LUTs and shifts\nchannel_luts = tttrlib.MapIntVectorFloat()\nchannel_luts[0] = lut_ch0\nchannel_luts[8] = lut_ch8\n\nchannel_shifts = tttrlib.MapSignedCharInt()\nchannel_shifts[0] = 0\nchannel_shifts[8] = 5\n\nresult_config = tttr.apply_channel_luts(channel_luts, channel_shifts)\nprint(f\"Configuration applied to {result_config} events\")\n\n# Apply linearization with specified seed\nprint(\"\\nApplying linearization...\")\nresult = tttr.apply_luts_and_shifts(42)  # Random seed for reproducibility\n\nif result >= 0:\n    print(\"Linearization successful!\")\nelse:\n    print(\"Linearization failed!\")\n\n# Get microtime histogram after linearization\nhist_after, bins_after = tttr.get_microtime_histogram()\nprint(f\"Microtime histogram after linearization: {len(hist_after)} bins\")"

        "# Summary\n\nThe microtime linearization functionality provides:\n\n1. **Per-channel configuration**: Direct LUT and shift assignment per routing channel\n2. **Dithering**: Random number-based dithering reduces quantization artifacts\n3. **Reproducibility**: Seed control for reproducible results\n4. **Flexibility**: Can apply LUTs, shifts, or both\n5. **Performance**: Optimized single-pass processing\n6. **Scalability**: Supports any number of channels with independent parameters\n\nChannel configuration:\n- Channels 0-255 supported\n- Each channel can have its own LUT and shift\n- LUTs must all have the same size (validated automatically)\n- Shifts are applied modulo the LUT size or default range\n\nFor more information about TAC linearization, see the Seidel Software\ntaclin algorithm documentation.\n\n"
