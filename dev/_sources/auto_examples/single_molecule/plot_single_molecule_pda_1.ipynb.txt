        "\n# Photon distribution analysis - 1\n\n## Photon distribution analysis\n\n### Theory\nThe library tttrlib provides the option to compute models for single molecule\nhistograns using Photon Distribution Analysis (PDA) for scoring of realizations\nof models by experimental single molecule histograms :cite:`kalinin2007`.\n\n#.. plot:: ../examples/single_molecule/single_molecule_pda_1.py\n\nThe ``tttrlib.Pda`` class implements the Pda algorithm as previously described\n:cite:`antonik2006`.\n\n### tttrlib.Pda objects\nTo create a PDA model first create an instance of the ``tttrlib.Pda`` class. The\nPda class implements a Photon Distribution Analysis algorithm for photons that\nare registed in two detection channels (channel 1 and channel 2). These two channels\ncan be any channel type. In a FRET experiment channel 1 (ch1) and channel 2 (ch2)\nare typically the green and red detection channel, respectively. In an anisotropy\nPDA experiment the two channels can be the parallel and perpendicular detection\nchannel. Using the theoretical probability for detecting a photon in the first\ndetection channel as an input (in the original manuscript this is called pG) the\nnumber of photons in the first and the second detection channel are computed.\nHereby, the algorithm considers the background fluorescence in the two detection\nchannels. The measured signal consists of fluorescence (F) and background (B)\ncontributions (S=F+B).\n\nTo compute a PDA histogram, first, create an instance of the  ``tttrlib.Pda`` class.\nWhen you create a new instance, you can specify a set of parameters that are of\nrelevance.\n\n```python\nimport tttrlib\nn_photons = 50\nkw = {\n    \"hist2d_nmax\": n_photons,\n    \"hist2d_nmin\": 10,\n    \"background_ch1\": 2.3,\n    \"background_ch2\": 1.2,\n}\npda = tttrlib.Pda(**kw)\n```\nThe ``Pda`` class will compute a matrix here called ``s1s2`` that will contain the\nprobabilities of detecting a photon in ch1 and ch2. The indices of the matrix correspond\nto the number of photons, the matrix values to the probability. The parameter\n``hist2d_nmax`` specifies up to wich maximum number of photons this matrix is\ncomputed. The parameter ``hist2d_nmin`` specifies the minimum number of photons in\nthis matrix which will be considered in later steps. The parameter ``background_ch1``\nand ``background_ch2`` specify the background count rate in the two channels.\n\nThe parameters can also be changed after the Pda object is created.\n\n```python\npda.background_ch1 = 2.0\npda.background_ch2 = 5.0\npda.hist2d_nmin = 5\npda.hist2d_nmax = 60\n```\nTo compute the PDA histogram intensity distribution of the fluorescence p(F),\nneeds to be specified. The intensity distribution of the fluorescence, P(F), can\nbe obtained from the total measured signal intensity distribution P(S) by deconvolution\nassuming that the background signals obey Poisson distributions. In this description\nwe simply compute a Poisson distribution for p(F).\n\n```python\nimport scipy.stats\nmu = 20 # expectation value for the number of photons\ndist = scipy.stats.poisson(mu)\nx = np.arange(0, n_photons)\npF = dist.pmf(x)\npda.setPF(pF)\n```\nThe last statement in the code above assigns the distribution p(F) to the Pda\nobject.\n\nNext, a set of species with associated amplitudes with corresponding theoretical\nprobabilities of detecting a photon in the first channel. This can be done by\neither assigning the amplitudes and the probabilities separately to the Pda object\n\n```python\namplitudes = [0.5, 0.5]\nprobabilities_ch1 = [0.8, 0.2]\npda.set_amplitudes(amplitudes)\npda.set_probabilities_ch1(probabilities_ch1)\n```\nor by assigning a spectrum that consists of interleaved amplitudes and probabilities\n\n```python\np_spectrum_ch1 = np.dstack([amplitudes, probabilities_ch1]).flatten()\npda.spectrum_ch1 = p_spectrum_ch1\n```\nThe spectrum is interleaved array [a1, p1, a2, p2, ...] where ai refers to\namplitudes and p1 refers to probabilities of registering a photon in the first\nchannel. The probabilities ``probabilities_ch1`` are the  theoretical probability\nof registering a photon in the first channel. In a FRET experiment the probability\nrelates to the FRET efficiency by\n\n\\begin{align}p_G = \\left( 1 + \u0007lpha + \frac{\\gamma E}{(1-E)} \right)^{-1}      ext{ with } \\gamma = \frac{g_R \\Phi_A}{g_R \\Phi_D}\\end{align}\n\nwhere $g_G$, $g_R$ are the detection efficiencies in the green and\nred detection channel, respectively. $\\Phi_A$, $\\Phi_D$ are the\nfluorescence quantum yield of the acceptor and the donor, respectively. $\u0007lpha$\nis the cross-talk from the donor to the acceptor channel, and $E$ is the\nFRET efficiency.\n\nThe computed distribution of photons in ch1 and ch2 is accessed by the attribute\n``s1s2``.\n\n```python\ns1s2 = pda.s1s2\n```\nThe matrix ``s1s2`` is computed when the attributed is accessed. The matrix is only\nupdated if a parameter of relevance is changed and the matrix is accessed.\n\nIn a PDA 2D analysis this matrix is often reduced in dimensionality to represent\nthe model and score against the data. For this dimensionality reduction\n``tttrlib.Pda`` offers a method. However, first, it needs to be specified how the\nmatrix is reduced in dimensionality. For that, a function needs to be specified\nand assigned to the object. Any python function with a least two arguments can\nbe used for that. The first argument always corresponds to ch1, the second\nargument to ch2. For instance, a histogram of the proximity ration can be computed\nby first defining a corresponding function and then creating a histogram using the\nmethod ``get_1dhistogram``.\n\n```python\ns1s2 = pda.s1s2\n# A one dimensional representation of the s1s2 matrix if obtain\n# by a function that projects the pairs of photons. Any python function\n# accepting at least two arguments can be used\nproximity_ratio = lambda ch1, ch2: ch2 / (ch1 + ch2)\n\n# The python function is used to set the attribute `histogram_function`\npda.histogram_function = proximity_ratio\n\n# The method get_1dhistogram of the Pda object returns a 1D histogram\n# of the s1s2 array for the specified function\nx_pr, y_pr = pda.get_1dhistogram(\n    log_x=False,\n    x_min=0.0,\n    x_max=1.0,\n    n_bins=21\n)\n```\nThe arguments of ``get_1dhistogram`` define the range and the resolution of the\nhistogram.\n\nFunctions, e.g., the FRET efficiency, that require additional parameters can be\npassed to the Pda object by defining a function with additional arguments. Note,\npotential division by zero need to be handled.\n\n```python\ndef fret_efficiency(ch1, ch2, phiD=0.8, phiA=0.32, det_ratio=0.32):\n    return 1.0 / (1. + phiD / phiA * det_ratio * ch2 / ch1)\n\npda.histogram_function = fret_efficiency\nx_eff, y_eff = pda.get_1dhistogram(\n    log_x=False,\n    x_min=0.0,\n    x_max=1.0,\n    n_bins=31\n)\n```\nHistograms with a logarithmic scale are computed by setting `log_x` to True.\nWhen the option ``skip_zero_photon`` is set to False the first column and row of\nthe s1s2 matrix (zero photons in ch1 or ch2) is used. In this case potential division\nby zeros in the histogram function need to be handled. The default value for\n``skip_zero_photon`` is True.\n\n```python\nsg_sr = lambda ch1, ch2: max(1, ch1) / max(1, ch2)\npda.histogram_function = sg_sr\nx_sgsr, y_sgsr = pda.get_1dhistogram(\n    log_x=True,\n    x_min=0.05,\n    x_max=80.0,\n    n_bins=31,\n    skip_zero_photon=False\n)\n```\nFinally, the 2D counting histogram and the 1D representations can be plotted.\n\n```python\nfig, ax = p.subplots(nrows=1, ncols=3)\nax[0].imshow(s1s2)\nax[1].plot(x_pr, y_pr, label='Proximity ratio')\nax[1].plot(x_eff, y_eff, label='FRET efficiency')\nax[1].legend()\nax[2].semilogx(x_sgsr, y_sgsr, label='Sg/Sr')\nax[2].legend()\np.show()\n```\n<div class=\"alert alert-info\"><h4>Note</h4><p>To score models against the data either the 2D histogram or the 1D representation\n    can be used. The scoring is described elsewhere :cite:`kalinin2007`.</p></div>\n\n### Example analysis of experimental data\nIn this section a small single-molecule dataset will be loaded and analyzed with\nby optimizing a set of parameters to experimental histograms using the ``tttrlib.Pda``\nclass and basic [SciPy](https://www.scipy.org/) functions. The photons that\ncorrespond to the data selected for the PDA histograms are indexed. This way,\nfluorescence decay curves that correspond to the PDA histograms are computed.\n\n#.. plot:: ../examples/single_molecule/single_molecule_pda_2.py\n\nThe first step when a dataset is analyzed by the ``tttrlib.Pda`` class is to load\nand create a ``TTTR`` object. In the example, the tttr data is split into multiple\nBH132 spc files. Hence, for simpler analysis, the data is first stacked into a single\n``TTTR`` object.\n\n```python\n# open a set of files and stack them in a single TTTR object\nimport os; from pathlib import Path\nDATA_ROOT = Path(os.environ.get(\"TTTRLIB_DATA\", \".\"))\nfiles = glob.glob(str(DATA_ROOT / 'bh/bh_spc132_smDNA/*.spc'))\ndata = tttrlib.TTTR(files[0], 'SPC-130')\nfor d in files[1:]:\n    data.append(tttrlib.TTTR(d, 'SPC-130'))\n```\nNext, the photons registered in the different routing channels need to be selected\nand the photon trace is split into time windows (tws). Time windows with a specified\nminimum number of photons are discriminated to reduce the background contribution to\nthe PDA histogram. The photons in the two detection channels are counted and a matrix\nS1S2 that contains a histogram of the photon counts is created. This photon count\nmatrix is computed up to a maximum number of photons that is specified ahead. The\nlarger the maximum number of photons in the S1S2 matrix, the slower the algorithm\nwill be. This maximum number depends on the size of the tws. For large tws the maximum\nnumber should be increased. For setups with large foci a larger maximum number of\nphotons should be used. Overall, the maximum number of photons should be adjusted\nto the experiment. By inspecting a S1S2 matrix this number can be adjusted. Moreover,\nThe distribution of the signal intensity P(S) needs to be determined. The PDA\nalgorithm requires the distribution of the fluorescence intensity P(F), which can\nbe determined from P(S). In practice, at low background when tws with low photons\ncounts are discriminated P(F) can be approximated by P(S). Moreover, for a consistent\nanalysis over multiple representations of the experimental data, e.g., TCSPC or FCS,\ncare must be taken that the tw selection did not introduce a selection bias or that\nthe selections are at least consistent.\n\nTo create a experimental P(S1,S2) histogram, that is stored in form of a matrix,\nthe static method ``tttrlib.Pda.compute_experimental_histograms`` is used. The\nmethod takes the channel selection, the tw size, and the minimum number of photons\nin a tw as an input.\n\n```python\n# define what is PDA channel 1 and channel 2\nchannels_1 = [0, 8]  # 0, 8 are green channels\nchannels_2 = [1, 9]  # 1,9 are red channels\nminimum_number_of_photons = 20\nmaximum_number_of_photons = 150\nminimum_time_window_length = 1.0\n\ns1s2_e, ps, tttr_indices = tttrlib.Pda.compute_experimental_histograms(\n    tttr_data=data,\n    channels_1=channels_1,\n    channels_2=channels_2,\n    maximum_number_of_photons=maximum_number_of_photons,\n    minimum_number_of_photons=minimum_number_of_photons,\n    minimum_time_window_length=minimum_time_window_length\n)\n```\nThe method ``tttrlib.Pda.compute_experimental_histograms`` returns the experimental\nhistogram that contains the counts in the first and second PDA channel, the histogram\nover the number of counts P(S), and the tttr indices as numpy arrays. These indices\ncan be used in later steps to process the associated photons.\n\nNext, a new instance of the ``tttrlib.Pda`` class needs to be created and a function\nthat converts P(S1,S2) into a 1 dimensional histogram needs to be specified.\n\n```python\n# define a Pda object\nkw_pda = {\n    \"hist2d_nmax\": maximum_number_of_photons,\n    \"hist2d_nmin\": minimum_number_of_photons,\n    \"pF\": ps\n}\npda = tttrlib.Pda(**kw_pda)\n# set a function to make a 1D histogram\npda.histogram_function = lambda ch1, ch2: ch1 / max(1, ch2)\n```\nHere, the ratio of the first and the second channel is used. Above we defined the\nassociated the first channel to the routing channel numbers [0, 8] and the second\nchannel to the routing channel numbers [1, 9]. For the dataset in this example this\ncorresponds to the \"green\" and \"red\" detection channel. Hence, the function used\nto create one dimensional representations of the P(S1,S2) matrix is the ratio of\nthe green and red signals.\n\nNow, we can define a set of parameters that is optimized to the experimental data.\nHere we use a three species model. Moreover, we define an initial values for the\ngreen and red background signal. Using these values, we compute a one dimensional\nhistogram weighted deviations of the model histogram for initial values.\n\n```python\ninitial_background_ch1 = 1.7\ninitial_background_ch2 = 0.7\ninitial_amplitudes = [0.33, 0.33, 0.33]\ninitial_probabilities_ch1 = [0.0, 0.35, 0.45]\nkw_hist = {\n    \"x_max\": 100.0,\n    \"x_min\": 0.1,\n    \"log_x\": True,\n    \"n_bins\": 81,\n    \"n_min\": 10\n}\n# get a initial model histogram\nx_model_initial, y_model_initial = pda.get_1dhistogram(\n    amplitudes=initial_amplitudes,\n    probabilities_ch1=initial_probabilities_ch1,\n    **kw_hist\n)\n# get the data histogram\nx_data, y_data = pda.get_1dhistogram(\n    s1s2=s1s2_e.flatten(),\n    **kw_hist\n)\ninitial_wres = (y_data - y_model_initial) / np.sqrt(y_data)\n```\nThe computation of the initial weighted residuals can be omited and is merely here\nfor instructive purposed. Here, the deviations are weighted by the number of counts\nin each histogram bin.\n\nNext, an objective function, i.e., a function that with a minimum value for parameters\nthat agree optimally with the data is defined. A simple objective function is the\nchi2 that measured the sum of the weighted squared deviations is used.\n\n```python\ndef chi2(\n    x0: np.ndarray,\n    y_data: np.ndarray,\n    pda_object: background_ch2 = x0[n_species * 2 + 1]\n    pda_object.background_ch1 = background_ch1\n    pda_object.background_ch2 = background_ch2\n    x_model, y_model = pda_object.get_1dhistogram(\n        amplitudes=amplitudes,\n        probabilities_ch1=probabilities,\n        **kw_hist\n    )\n    weights = np.sqrt(y_data)\n    np.place(weights, weights == 0, 10000000.0)\n    wres = (y_data -
