        "\n# Reading of CLSM data\nIn laser scanning microscopy (LSM) with time-tagged time resolved (TTTR)\ndetection, the LSM image is stored in a stream of events. Events are either\nphotons or special events that can for instance mark the beginning of a\nnew frame, a new  line in laser scanning, or the end of a line.\n``tttrlib`` uses a special LSM container that maps the photon events in a\nTTTR stream to pixels in an image. When a LSM container is constructed the\nTTTR stream is read and the beginning and the end of a frame, line, and pixel\nare identified.\n\nA CLSM image container contains a list of frames. Each frame contains a list of\nlines and each line contains a list of pixel. Frames, lines, and pixels refer to\na photon index ranges in the TTTR data stream. When a new CLSMImage container\nis created the markers for frames and lines contained in the data stream are read.\nThere are usually no pixel markers. Markers are identified in the\ndata stream by numbers. There are no default identifier for frames and lines\n(see `auto_examples/flim/plot_marker.py`). Nevertheless, if no identifier\nfor markers are provided, either the default values for a particular setup are used,\nor meta-data contained in the TTTR file is read.\n"
